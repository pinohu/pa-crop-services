# SuiteDash Custom Fields — Setup Guide
## PA CROP Services — Required Custom Fields

These 6 custom fields must be created in SuiteDash for the PA CROP portal and 
lead scoring system to work correctly.

### How to Create Custom Fields in SuiteDash

1. Log into SuiteDash → https://app.suitedash.com
2. Click the gear icon (⚙️) in the left sidebar → Settings
3. Navigate to: CRM → Custom Fields → Contacts
4. Click "Add Custom Field" for each field below

---

## Fields to Create

### 1. portal_access_code
- **Field Key**: portal_access_code
- **Label**: Portal Access Code
- **Type**: Text (single line)
- **Required**: No
- **Notes**: Auto-populated by n8n onboarding workflow when client pays

### 2. lead_score
- **Field Key**: lead_score
- **Label**: Lead Score
- **Type**: Number
- **Required**: No
- **Notes**: Set by /api/intake based on behavioral signals (0-100)

### 3. lead_tier
- **Field Key**: lead_tier
- **Label**: Lead Tier
- **Type**: Text (single line)
- **Required**: No
- **Notes**: Values: "cold", "warm", "hot" — set by /api/intake

### 4. lead_source
- **Field Key**: lead_source
- **Label**: Lead Source
- **Type**: Text (single line)
- **Required**: No
- **Notes**: Examples: "website", "partner-widget", "compliance-check", "newsletter"

### 5. entity_type
- **Field Key**: entity_type
- **Label**: Entity Type
- **Type**: Text (single line)
- **Required**: No
- **Notes**: PA entity type: llc, corp, lp, foreign, etc.

### 6. mobile_phone
- **Field Key**: mobile_phone
- **Label**: Mobile Phone
- **Type**: Phone
- **Required**: No
- **Notes**: Used for SMS backup notifications on annual report reminders

### 7. crop_plan
- **Field Key**: crop_plan
- **Label**: CROP Plan
- **Type**: Text (single line)
- **Required**: No
- **Notes**: Values: compliance_only, business_starter, business_pro, business_empire

### 8. crop_since
- **Field Key**: crop_since
- **Label**: CROP Service Since
- **Type**: Date
- **Required**: No
- **Notes**: Date client activated service

### 9. referral_code
- **Field Key**: referral_code
- **Label**: Referral Code
- **Type**: Text (single line)
- **Required**: No
- **Notes**: Auto-generated by onboarding: CROP-[last6ofContactID]

### 10. has_foreign_entity
- **Field Key**: has_foreign_entity
- **Label**: Has Foreign Entity?
- **Type**: Text (single line)
- **Required**: No
- **Notes**: yes/no — from compliance check or intake form

---

## After Creating Fields

Once these fields exist in SuiteDash, the following will automatically work:
- Portal login via /api/auth will read portal_access_code
- n8n onboarding workflow will write all fields on Stripe payment
- Lead scoring via /api/intake will write lead_score and lead_tier
- Annual report reminder sequence can filter by crop_plan
- Partner attribution visible via referral_code

## Verification

To verify fields are working, check a contact record in SuiteDash after a test
Stripe payment — you should see all fields populated.
